## Options

### `--config`

<div className="mt-3 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   Path to your auth config file, relative to the project root. Use this if your config is not in
   one of the default locations.
</div>

<div className="mt-5 rounded-xl border border-border bg-card overflow-hidden">
   <CodeBlock lang="bash" code={`npx @swift-auth/cli generate --config ./src/lib/auth.ts`} />
</div>

### `--output`

<div className="mt-3 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   Path where the generated schema file should be written, relative to the project root. If a file
   already exists at this path, its contents will be overwritten.
</div>

<div className="mt-5 rounded-xl border border-border bg-card overflow-hidden">
   <CodeBlock lang="bash" code={`npx @swift-auth/cli generate --output ./src/db/auth-schema.ts`} />
</div>

<div className="mt-4 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   Both options can be combined.
</div>

<div className="mt-5 rounded-xl border border-border bg-card overflow-hidden">
   <CodeBlock
      lang="bash"
      code={`npx @swift-auth/cli generate --config ./src/lib/auth.ts --output ./src/db/auth-schema.ts`}
   />
</div>

## Config file detection

<div className="mt-3 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   When <code>--config</code> is not provided, the CLI looks for your auth config in these locations
   automatically.
</div>

<div className="mt-5 rounded-xl border border-border bg-card overflow-hidden">
   <CodeBlock
      lang="bash"
      code={`./src/lib/auth.ts
./src/lib/auth.js
./lib/auth.ts
./lib/auth.js`}
   />
</div>

<div className="mt-4 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   If your config is outside these paths, pass it explicitly with <code>--config</code>.
</div>
