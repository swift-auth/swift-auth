# Email & Password

<div className="mt-4 max-w-[560px] text-lg leading-relaxed text-muted-foreground">
   Authio has built-in email and password authentication with support for email verification, auto
   sign-in, password reset, and configurable password rules.
</div>

## Basic setup

<div className="mt-3 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   To enable email and password auth, add <code>emailAndPassword</code> to your auth config with{' '}
   <code>enabled: true</code>.
</div>

<div className="mt-5 rounded-xl border border-border bg-card overflow-hidden">
   <div className="border-b border-border px-5 py-2.5">
      <span className="font-mono text-xs text-muted-foreground">src/lib/auth.ts</span>
   </div>
<CodeBlock
   lang="ts"
   code={`import { Authio } from '@authio/core';
import { drizzleAdapter } from '@authio/drizzle';
import { db } from '../db/index.js';

export const auth = new Authio({
  baseUrl: '<http://localhost:8000>',
  emailAndPassword: {
    enabled: true,
  },
  database: drizzleAdapter({
    db,
    provider: 'postgres', // 'postgres' | 'mysql' | 'sqlite'
  }),
});`}
/>
</div>

## Configuration options

<div className="mt-3 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   Here is everything you can configure under <code>emailAndPassword</code>.
</div>

<div className="mt-5 rounded-xl border border-border bg-card overflow-hidden">
  <CodeBlock
    lang="ts"
    code={`emailAndPassword: {
  enabled: true,

// automatically sign the user in after signup (default: true)
autoSignIn: true,

// require users to verify their email before signing in (default: false)
verifyEmail: false,

// minimum password length (default: 8)
minPasswordLength: 8,

// how long the verification or reset token is valid in ms (default: 1 hour)
verificationTokenExpiry: 1000 _60_ 60,

// called after signup when verifyEmail is true — send the token however you want
verificationCallback: async ({ name, email, verificationToken }) => {},

// called when a user requests a password reset — send the token however you want
forgotPasswordCallback: async ({ name, email, resetToken }) => {},
}`}
/>

</div>

## Email verification

<div className="mt-3 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   When <code>verifyEmail</code> is <code>true</code>, Authio creates the user but does not sign
   them in. Instead it calls your <code>verificationCallback</code> with a token. You decide how to
   deliver it — email, SMS, anything.
</div>

<div className="mt-4 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   The <code>verificationCallback</code> is required when <code>verifyEmail</code> is enabled —
   Authio will throw at startup if it is missing.
</div>

<div className="mt-5 rounded-xl border border-border bg-card overflow-hidden">
   <div className="border-b border-border px-5 py-2.5">
      <span className="font-mono text-xs text-muted-foreground">src/lib/auth.ts</span>
   </div>
   <CodeBlock
      lang="ts"
      code={`emailAndPassword: {
  enabled: true,
  verifyEmail: true,
  verificationCallback: async ({ name, email, verificationToken }) => {
    // send the token to the user however you want
    await sendEmail({
      to: email,
      subject: 'Verify your email',
      body: \`Hi \${name}, your verification token is: \${verificationToken}\`,
    });
  },
},`}
   />
</div>

<div className="mt-6 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   Once the user has the token, call <code>verifyEmail</code> from your frontend to complete
   verification. If <code>autoSignIn</code> is enabled, a session is created automatically after
   verification.
</div>

<div className="mt-5 rounded-xl border border-border bg-card overflow-hidden">
   <CodeBlock lang="ts" code={`await authClient.verifyEmail(token);`} />
</div>

## Password reset

<div className="mt-3 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   To enable password reset, provide a <code>forgotPasswordCallback</code>. Authio generates a reset
   token and passes it to your callback — you deliver it however you want.
</div>

<div className="mt-4 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   If no account exists for the given email, Authio returns a success response anyway. This prevents
   leaking which emails are registered.
</div>

<div className="mt-5 rounded-xl border border-border bg-card overflow-hidden">
   <div className="border-b border-border px-5 py-2.5">
      <span className="font-mono text-xs text-muted-foreground">src/lib/auth.ts</span>
   </div>
   <CodeBlock
      lang="ts"
      code={`emailAndPassword: {
  enabled: true,
  forgotPasswordCallback: async ({ name, email, resetToken }) => {
    await sendEmail({
      to: email,
      subject: 'Reset your password',
      body: \`Hi \${name}, your reset token is: \${resetToken}\`,
    });
  },
},`}
   />
</div>

<div className="mt-6 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   From the frontend, first request a reset then submit the new password with the token.
</div>

<div className="mt-5 rounded-xl border border-border bg-card overflow-hidden">
  <CodeBlock
    lang="ts"
    code={`// request a reset token
await authClient.forgotPassword(email);

// reset the password using the token
await authClient.resetPassword(token, newPassword);`}
/>

</div>

<div className="mt-4 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   After a successful password reset, all existing sessions for that user are invalidated for
   security.
</div>

## Frontend usage

<div className="mt-3 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   All email and password methods are available on the auth client.
</div>

<div className="mt-5 rounded-xl border border-border bg-card overflow-hidden">
  <CodeBlock
    lang="ts"
    code={`// sign up
await authClient.emailSignUp({ name, email, password });

// sign in
await authClient.emailSignIn({ email, password });

// sign out
await authClient.signOut();

// get current session
await authClient.getSession();`}
/>

</div>
