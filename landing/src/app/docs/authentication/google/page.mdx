## 3. Add the provider to your auth config

<div className="mt-3 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   Import <code>googleProvider</code> from <code>swift-auth/providers</code> and add it to{' '}
   <code>socialProviders</code> in your config.
</div>

<div className="mt-5 rounded-xl border border-border bg-card overflow-hidden">
  <div className="border-b border-border px-5 py-2.5">
    <span className="font-mono text-xs text-muted-foreground">src/lib/auth.ts</span>
  </div>
  <CodeBlock
    lang="ts"
    code={`import { SwiftAuth } from 'swift-auth';
import { drizzleAdapter } from '@swift-auth/drizzle';
import { db } from '../db/index.js';
import { googleProvider } from 'swift-auth/providers';

export const auth = new SwiftAuth({
  baseUrl: '<http://localhost:8000>',
  database: drizzleAdapter({
    db,
    provider: 'postgres', // 'postgres' | 'mysql' | 'sqlite'
  }),
  socialProviders: {
    google: googleProvider({
      clientId: process.env.GOOGLE_CLIENT_ID!,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
      redirectUrl: '<http://localhost:3000>',
    }),
  },
});`}
  />
</div>

<div className="mt-4 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   The <code>redirectUrl</code> is your frontend URL — where the user lands after the OAuth flow
   completes. This is not the Google callback URL, Swift Auth handles that internally.
</div>

## Configuration options

<div className="mt-3 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   Beyond the required fields, you can customize the consent screen behavior and the scopes you
   request from Google.
</div>

<div className="mt-5 rounded-xl border border-border bg-card overflow-hidden">
  <CodeBlock
    lang="ts"
    code={`googleProvider({
  clientId: process.env.GOOGLE_CLIENT_ID!,
  clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
  redirectUrl: 'http://localhost:3000',

// scopes to request from Google (default: ['openid', 'email', 'profile'])
scopes: ['openid', 'email', 'profile'],

// controls the consent screen behavior (default: 'consent')
prompt: 'consent',

// 'offline' returns a refresh token, 'online' does not (default: 'offline')
accessType: 'offline',
})`}
/>

</div>

### scopes

<div className="mt-3 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   Scopes control what data your app can access from Google. The defaults — <code>openid</code>,{' '}
   <code>email</code>, and <code>profile</code> — are enough to get the user's name, email, and
   profile picture. If your app needs access to other Google services like Calendar or Drive, add
   those scopes here. Any extra scopes will be shown to the user on the consent screen.
</div>

### prompt

<div className="mt-3 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   Controls what Google shows the user when they sign in. The available values are:
</div>

<div className="mt-4 max-w-[560px] space-y-3">
   <div className="flex gap-3 text-sm">
      <code className="shrink-0 rounded bg-muted px-1.5 py-0.5 font-mono text-xs">consent</code>
      <span className="text-muted-foreground">
         Always show the consent screen, even if the user has already approved. This is the default
         and ensures a refresh token is returned.
      </span>
   </div>
   <div className="flex gap-3 text-sm">
      <code className="shrink-0 rounded bg-muted px-1.5 py-0.5 font-mono text-xs">
         select_account
      </code>
      <span className="text-muted-foreground">
         Show the account picker so the user can choose which Google account to use.
      </span>
   </div>
   <div className="flex gap-3 text-sm">
      <code className="shrink-0 rounded bg-muted px-1.5 py-0.5 font-mono text-xs">login</code>
      <span className="text-muted-foreground">
         Force the user to re-enter their Google credentials even if they are already signed in.
      </span>
   </div>
   <div className="flex gap-3 text-sm">
      <code className="shrink-0 rounded bg-muted px-1.5 py-0.5 font-mono text-xs">none</code>
      <span className="text-muted-foreground">
         Skip the consent screen if the user has already approved. Returns an error if they haven't.
      </span>
   </div>
</div>

### accessType

<div className="mt-4 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
   When set to <code>offline</code> (the default), Google returns a refresh token along with the
   access token. This lets your server refresh access without the user signing in again. Set to{' '}
   <code>online</code> if you only need a short-lived access token and don't need offline access.
</div>

## How Swift Auth handles the flow

<div className="mt-3 max-w-[560px] text-sm leading-relaxed text-muted-foreground">
